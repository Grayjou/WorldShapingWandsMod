## Project Overview for New Agents / Engineers

WorldShapingWandsMod is a tModLoader mod for Terraria that provides 10 wand families
(Building, Dismantling, Replacement, Wiring, Safekeeping, Coating, Fluids, Torches,
Delimitation, Molding) × 4 selection modes (Instant, Select, Confirm, Stamp) = 40 items,
plus 2 standalone TorchWheel wands, enabling players to edit the world in
geometric shapes (Rectangle, Ellipse, Diamond, Triangle, Elbow, Cardinal Line, Straight
Line — plus half-shapes via SliceMode) with configurable Filled/Hollow modes.

### Key Design Patterns
- Provider pattern: shapes implement IShapeProvider; ShapeRegistry maps enum → provider
- Immutable state: SelectionState and ShapeContext are immutable (factory methods)
- Per-wand settings: each family has its own struct on WandPlayer — no global state
- Client-side safety: all rendering is [Autoload(Side = ModSide.Client)]
- Server-authoritative MP: WandPacketHandler handles client → server → broadcast

### Localization Workflow (Translator-Friendly Round-Trip)

For bulk translation work (e.g. when GrayJou wants to take a batch of untranslated keys
to a spreadsheet, translate them, and import the result back), use:

```
python Scripts/Localization/locale_spreadsheet.py --list-pending             # per-locale untranslated counts
python Scripts/Localization/locale_spreadsheet.py --export-pending           # TSV with ONLY untranslated rows
python Scripts/Localization/locale_spreadsheet.py --export-pending --locale es-ES   # one-language pass
# (translate cells in the spreadsheet, save TSV)
python Scripts/Localization/locale_spreadsheet.py --import --from Localization/_spreadsheet_pending.tsv
```

Single-key edits should still be done by hand-editing the relevant `.hjson` file directly
or via `Scripts/update_localization.py` — the spreadsheet round-trip is for bulk passes.

### Placeholder Asset Convention (CRITICAL — MUST FOLLOW)

**When wiring a NEW button or UI element to a texture, ALWAYS reference a unique,
previously-unused asset path** — even if the PNG doesn't exist yet. NEVER reuse an
existing icon path for a new button purely as a "placeholder until art lands."

- ✅ Correct: `ModContent.Request<Texture2D>("WorldShapingWandsMod/Assets_Build/Icons/<Family>/Placeholders/<DescriptiveName>")`
  (tModLoader renders a pink-checkerboard "missing texture" sprite at runtime; that
  IS the placeholder, and the user can drop a real PNG at the same path later or use
  texture-override against the unique path without affecting other buttons.)
- ❌ Incorrect: pointing the new button at `Assets/Icons/Building/Actuation.png`
  "for now". This silently couples the new button to an unrelated icon, makes
  texture-override nuke both, and gives no visual cue that art is pending.

When a new button is shipped, the patch notes / Implemented log MUST state the chosen
asset path so the user has a paper trail. This convention was made permanent on
2026-04-23 (Letter #11) so future agents inherit it even after session resets.

### Naming Conventions (Lexicographic Ordering)

All date-based filenames use **YYYY-MM-DD** format for lexicographic sort:

| File Type | Format | Example |
|-----------|--------|---------|
| **Sessions** | `Sessions/YYYY-MM-DD Session N.txt` | `2026-04-15 Session 1.txt` |
| **Letters** | `YYYY-MM-DD Letter #N.md` (at Cloned root) | `2026-04-15 Letter #1.md` |
| **Implementation Logs** | `dev_notes/planning/Implemented_YYYY-MM-DD.md` | `Implemented_2026-04-15.md` |

Implementation logs use the **date of the first session** in that log file. Multiple sessions on the same day are sections within the same file (S1, S2, S3...). New days get new files.

Legacy files (ImplementedSession9.md through ImplementedSession16.md) are kept as-is — only new logs use the date format.
