================================================================================
lint_asset_paths.py — DESIGN RATIONALE
================================================================================

Per AgentFactory §4e: a build that compiles GREEN can still ship missing
textures, because tModLoader's asset resolution is runtime-lazy. A typo in
ModContent.Request<Texture2D>("MyMod/Items/MisspeltSword") will compile
and link cleanly, only to throw at first item-instantiation.

This script closes the gap between "code compiles" and "asset graph is
complete." It runs as the second half of the dual build-gate:

    GREEN  =  (dotnet build == GREEN)  AND  (lint_asset_paths.py == exit 0)

--------------------------------------------------------------------------------
SCOPE
--------------------------------------------------------------------------------
- Catches: typos in asset path strings; assets removed but still referenced;
  case-sensitivity mismatches on case-sensitive filesystems.
- Does NOT catch: assets that exist on disk but are wrong (e.g. wrong size,
  wrong format, blank). Those are a content-review concern.
- Does NOT catch: dynamically-built strings (e.g. $"{mod}/{kind}/{name}").
  By design — those need runtime testing anyway.

--------------------------------------------------------------------------------
WHY REGEX, NOT ROSLYN
--------------------------------------------------------------------------------
- A Roslyn analyser would catch dynamically-built strings only when they
  could be constant-folded. The marginal coverage is small.
- Regex is a single 80-line .py file with no external dependencies.
- Regex matches every form including assignments and SoundStyle ctors,
  whereas a single Roslyn rule needs separate handlers per call shape.

--------------------------------------------------------------------------------
WHY EXIT CODES, NOT EXCEPTIONS
--------------------------------------------------------------------------------
- The script is part of a build gate. CI / dotnet build wrappers expect a
  non-zero exit on failure. Exception output is for humans only.

--------------------------------------------------------------------------------
WHY MOD_INTERNAL_NAME = ROOT.name
--------------------------------------------------------------------------------
- 99% of mods set <AssemblyName> equal to the directory name. If yours
  doesn't, hard-code it at the top of the script. We deliberately do NOT
  parse the .csproj — that's a fragile dependency for an 80-line linter.

--------------------------------------------------------------------------------
WHY ANY_EXTS, NOT TYPE-SPECIFIC RESOLUTION
--------------------------------------------------------------------------------
- tModLoader resolves "MyMod/Items/Sword" by trying .png, .xnb, .rawimg
  in order. The script does the same union resolution. We accept some
  false-positives (a SoundStyle pointing at a .png will not be flagged)
  in exchange for never needing to know the call site's expected type.
- This is a deliberate trade-off: the lint is a cheap "exists" check, not
  a type checker.
================================================================================
