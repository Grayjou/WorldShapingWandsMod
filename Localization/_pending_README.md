## Notes

- **You can fill these files in any order or partially.** A row left blank in
  the TSV is simply re-written as the same commented stub; nothing is broken.
- **Both TSVs are independent.** Importing one does not touch the other locale.
- These files are working artifacts; once imported they can be deleted (or kept
  as a working copy if you plan to come back to them). They are not currently
  in `.gitignore` — add a `Localization/_pending_*.tsv` line if you want them
  to stay out of source control.
- If you want a fresh export at any time (e.g. after new keys land in en-US),
  re-run the same `--export-pending --locale <code> --output …` command — the
  script will regenerate the TSV with the latest pending set.
